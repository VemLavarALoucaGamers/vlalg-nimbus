# nb-input-file
::BannerTest
::

File picker component for [Vue.js 3+](https://vuejs.org/). Uses a native `<input type="file">` (hidden) plus a themed control. Validation (accept, extensions, size, duplicates, optional GIF size / video rules) is shared with an internal helper; emits raw `File[]` for previews or uploads in the parent.

> **Note:** `NbInputFile` may not be registered on the default plugin export yet. Import the SFC directly from the package source in the monorepo or register it locally (see **Usage**).

## Installation

::code-group
  ```bash [Yarn]
  yarn add @vlalg-nimbus/nb-inputs
  ```
  ```bash [NPM]
  npm install @vlalg-nimbus/nb-inputs
  ```
::

## Usage

Register globally like other nb-inputs components, or import the `.vue` file in apps that ship the source:

::code-group
  ```vue [Local import (monorepo / source)]
  <script setup>
  import NbInputFile from '@vlalg-nimbus/nb-inputs/path/to/NbInputFile.vue'
  </script>

  <template>
    <NbInputFile
      nb-id="f1"
      input-name="upload"
      :show-label="true"
      label="Attachments"
      file-extension=".pdf,.png"
      :max-file-size-bytes="5 * 1024 * 1024"
      :multiple="true"
      :max-files="5"
      @changed="(files) => console.log(files)"
      @validation-error="({ file, message }) => console.warn(message, file)"
    />
  </template>
  ```
::

## NbFileUpload (upload queue only)

`NbInputFile` only picks and validates files; **`NbFileUpload`** is a separate component.

Documentação dedicada: `7.nb-file-upload.md`.

It sends each file with **`XMLHttpRequest`** + **`FormData`**, reports progress, supports parallel uploads (`concurrent`), retries (`maxRetries`, `retryDelayMs`), and cancel one or all active transfers.

It keeps row state: `pending` → `uploading` → `success` | `error` | `cancelled`.

It uses the same **`duplicateKey`** idea as validation when syncing with `files`, so reordered lists stay stable.

Original summary:

Register via the default `nb-inputs` plugin as **`NbFileUpload`**, or import the SFC from the package source.

Key props: `nbId`, `files`, `uploadUrl`, `concurrent`, `maxRetries`, `retryDelayMs`, `formFieldName`, `uploadMethod`, `requestHeaders`, `autoStart`, `showStartButton`, `minProgressDisplayMs`, `afterUploadMode`, `reuploadOnStart`, `uiLocale`.

- **`afterUploadMode`**: `clear-on-success` só actua quando **todas** as linhas estão `success` (um `cancelled` ou `error` impede limpar). `lock-until-clear-all` bloqueia o **Start** quando o lote terminou **e** houve **pelo menos um** `success` (ex.: mistura success+cancelled também bloqueia); use **Limpar tudo** ou lista vazia para desbloquear.
- **`reuploadOnStart`** (default `true`): cada clique em **Start** repõe `success` / `error` / `cancelled` → `pending` e volta a enviar **todos** os ficheiros da fila. Com `false`, só sobem linhas já `pending`.

Two-way sync: emita `update:files` com o novo `File[]` ao **limpar tudo** ou **remover um item** — use `v-model:files` (ou `:files` + `@update:files`) no pai. Se também usar `NbInputFile`, pode ser preciso remontar o picker (por exemplo `:key`) quando a lista ficar vinda só do upload, para a UI do input coincidir com o `File[]`.

Events: `upload-progress`, `upload-complete`, `upload-error`, `upload-all-complete`, `update:files`.

Exposed methods: `startUpload`, `cancelRow`, `cancelAllActive`, `clearAll`, `removeQueueItem`, `retryRow`, `retryAllErrors`.

**Progresso:** com ficheiros pequenos o browser costuma emitir um único salto 0→100 no XHR; `minProgressDisplayMs > 0` suaviza a barra. Atraso no servidor *depois* de receber o body não aumenta eventos de upload — use ficheiros maiores, rede lenta (DevTools), ou esta prop.

## Props

> Items with an (*) are required.

| name | Value type | Default | Description |
| ---- | :--------- | :------ | :---------- |
| nbId (*) | String | — | Root element id |
| inputName (*) | String | — | `name` / label `for` target |
| fileExtension | String | `''` | `accept` string (comma-separated types/extensions, e.g. `image/*`, `.pdf`) |
| fileExtensions | String | `''` | Legacy alias when `fileExtension` is empty |
| multiple | Boolean | false | Multiple files |
| maxFiles | Number, null | null | Max count when `multiple` |
| maxFileSizeBytes | Number | `5 * 1024 * 1024` | Max size per file (bytes) |
| allowedExtensions | Array | `[]` | Extra allowlist: `['.png','.jpg']` — applied **in addition** to `accept` when non-empty |
| allowDuplicates | Boolean | false | Allow same `name` + `size` again |
| gifWidth | Array | `[]` | Allowed GIF widths (pairs with `gifHeight`) |
| gifHeight | Array | `[]` | Allowed GIF heights |
| videoRatio | String | `''` | e.g. `16:9` — empty skips aspect check |
| videoMaxDuration | Number | null | Max duration in seconds (videos only) |
| videoAspectTolerance | Number | `0.01` | Tolerance for aspect ratio check |
| showFilesCounter | Boolean | true | Show counter when `multiple` and `maxFiles` is set |
| locale | Object | `{}` | Partial override for validation/counter strings (see **Locale keys**) |
| showConstraintsText | Boolean | true | Summary line (max files / max size) |
| multipleFilesSelectedText | String | `'files selected'` | Label when several files selected |
| hasIcon | Boolean | false | Leading/trailing icon area |
| iconDirection | String | `'left'` | `left` or `right` |
| hasTabIndexEnter | Boolean | true | Enter/Space on the visible control opens the picker |
| ariaLabel | String | … | Accessible name fallback |
| … | … | … | Theming / label / `inputStyle` props follow the same patterns as `NbInput` / docs in repo |

## Events

| name | Payload | Description |
| ---- | ------- | ----------- |
| changed | `File[]` | Current list after a successful selection merge |
| current-value | `File[]` | Same as `changed` |
| validation-error | `{ file: File \| null, message: string }` | Fires for each rejected file or unknown file when `maxFiles` is reached (`file` may be `null`) |
| focused / blurred | — | Label / control focus proxy |
| clicked | Event | Wrapper click |

## Locale keys

Pass only the keys you need on `locale`; others stay in English.

| key | Placeholders | Default (English) |
| --- | --- | --- |
| typeNotAllowed | `{fileType}`, `{accept}` | File type not allowed… |
| extensionNotAllowed | `{fileName}`, `{list}` | Extension not allowed… |
| sizeExceeded | `{fileName}` | File exceeds maximum size |
| duplicateInList | `{fileName}` | Already added |
| duplicateInBatch | `{fileName}` | Duplicate in same selection |
| gifDimensionsInvalid | `{width}`, `{height}`, `{allowedPairs}` | Invalid GIF dimensions |
| gifLoadError | — | GIF load error |
| videoAspectInvalid | `{width}`, `{height}`, `{aspect}`, `{ratio}` | Invalid aspect |
| videoDurationInvalid | `{duration}`, `{maxDuration}` | Duration too long |
| videoLoadError | — | Video load error |
| maxFilesReached | `{max}` | Limit reached |
| filesCounter | `{current}`, `{max}` | `{current}/{max}` |

Example:

```vue
<NbInputFile
  :locale="{
    filesCounter: '{current} de {max} arquivos',
    sizeExceeded: 'O arquivo \'{fileName}\' excede o tamanho máximo.',
    maxFilesReached: 'Máximo de {max} arquivo(s).'
  }"
/>
```

## Behaviour notes

- The visible field has **tab focus**; **Enter** / **Space** open the native dialog (`hasTabIndexEnter` gates activation).
- **GIF** and **video** checks run in the browser (image/video metadata); large files may add a short delay.
- **Remove** in the list uses the item **index** so files with the same name but different size are handled correctly.

## Limitations

- No built-in drag-and-drop on `NbInputFile`; pair it with **`NbFileUpload`** (or your own logic) for HTTP upload using the `File[]` from `@changed`.
- Internal validation helper is **not** part of the public npm API (not exported from `dist` as a separate module).
- Maintainer reference (detailed English): `packages/vue-components/nb-inputs/src/helpers/nbFileValidation.md`.
